# IMPROVEMENTS.md — улучшения MVP

## Реализованное улучшение

### Диагностический вывод источников в Streamlit

В базовом учебном RAG важно не только получить ответ, но и показать, на чем он основан. Поэтому в UI добавлен вывод источников:

- `doc_id`;
- `score`;
- название записи;
- полный текст найденного чанка.

Это улучшает проверяемость RAG: видно, какие документы попали в top-k и почему ответ считается релевантным.

## План следующих улучшений

### 1. Embeddings вместо TF-IDF

Сейчас поиск лексический: если в вопросе нет совпадающих слов, релевантный документ может не найтись. Следующий шаг — добавить `sentence-transformers` и искать по смысловым векторам.

Файлы для изменения:

- `scripts/build_index.py`;
- `app/retriever.py`;
- `pyproject.toml`.

Проверка результата: сравнить ответы на прямые и перефразированные вопросы.

### 2. Reranking

После первичного retrieval можно пересортировать top-k фрагменты более точной моделью. Это уменьшит шум в источниках.

Файлы для изменения:

- `app/retriever.py`;
- новый модуль `app/reranker.py`.

Проверка результата: добавить eval-набор вопросов и считать hit-rate@3.

### 3. Более строгий отказ

Сейчас отказ основан на пороге `MIN_SCORE`. Можно добавить проверку покрытия ключевых терминов вопроса и confidence label.

Файлы для изменения:

- `app/generator.py`;
- `app/prompts.py`;
- `tests/test_retrieval.py`.

Проверка результата: negative-вопросы не должны возвращать ответ по нерелевантным источникам.
