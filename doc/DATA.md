# DATA.md — описание данных

## Источник

Для учебного RAG используется открытая тематика и структура данных **Consumer Financial Protection Bureau Consumer Complaint Database** — публичной базы жалоб потребителей на финансовые организации.

В репозитории зафиксирован подготовленный учебный корпус `data/raw/datasets.json`: 1000 русскоязычных текстовых записей в формате, удобном для RAG. Каждая запись описывает жалобу потребителя: продукт, компанию, проблему, штат, сумму спора, срок обработки, статус ответа компании и ключевые термины.

## Почему выбран этот источник

CFPB Consumer Complaint Database хорошо подходит для RAG, потому что записи имеют текстовое описание, понятную предметную область и полезные метаданные. На таких данных легко проверить, что retrieval действительно находит релевантные фрагменты, а negative-вопросы не приводят к выдуманным ответам.

## Объем

- Исходный файл: `data/raw/datasets.json`
- Количество записей: **1000**
- Минимальный формат записи: `id`, `name`, `text`
- После chunking получается **1000 чанков**: по одному полному текстовому документу на запись при текущих настройках `CHUNK_MAX_CHARS = 1200` и `CHUNK_OVERLAP = 0`.

## Что индексируется

Индексируется только поле `text`.

| Поле | Индексируется | Назначение |
|---|---:|---|
| `id` | Нет | Преобразуется в `doc_id`, выводится в источниках |
| `name` | Нет | Название документа в UI |
| `text` | Да | Основной текст для TF-IDF retrieval |

## Темы корпуса

В корпусе представлены типовые финансовые продукты и проблемы:

- ипотека;
- студенческий кредит;
- кредитная карта;
- расчетный счет;
- взыскание долга;
- денежный перевод;
- краткосрочный займ;
- автокредит;
- предоплаченная карта;
- кредитная история.

## Pipeline обработки

```text
data/raw/datasets.json
  -> scripts/ingest.py
  -> data/processed/documents.jsonl
  -> app/chunker.py
  -> data/processed/chunks.jsonl
  -> scripts/build_index.py
  -> data/index/vectorizer.pkl + data/index/matrix.npz + data/index/chunks.jsonl
```

## Границы данных

Корпус предназначен для учебной демонстрации RAG-pipeline. Он не является юридической, финансовой или статистической аналитикой по CFPB. Цель — проверить end-to-end механику поиска, чанкинга, индексации, ответа и отображения источников.
